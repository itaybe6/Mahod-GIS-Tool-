# תכנון מסלול A→B (Route planner)

## איך לבדוק שהפיצ'ר עובד

1. פתחו באפליקציה את הנתיב **`/route-planner`** (או מהתפריט: «תכנון מסלול A→B»).
2. ודאו ש־**Supabase** מוגדר (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) וש־**Mapbox** מוגדר (`VITE_MAPBOX_ACCESS_TOKEN`) — החיפוש משלב כתובות (Mapbox) ותחנות מתוך **GTFS** (`gtfs_stops`), כדי שגם שמות כמו «ת. רכבת …» יופיעו בהשלמה אוטומטית.
3. הזינו **נקודת מוצא** ו**יעד**, והריצו **«מצא מסלול»**. מומלץ להתחיל עם זוגות שנבדקו מול המסד (קו ישיר אחד, מרחק הליכה סביר):

### דוגמאות מומלצות לחיפוש

| תפקיד | מה להקליד בשדה (או דומה) | הערה |
|--------|---------------------------|------|
| **מוצא (A)** | `קרליבך, תל אביב` או `קניון TLV, תל אביב` | קרוב לתחנה «קניון טי.אל.וי/קרליבך» |
| **יעד (B)** | `כיכר המדינה, תל אביב` או `ז'בוטינסקי כיכר המדינה` | קרוב לתחנה «ז'בוטינסקי/כיכר המדינה» |
| **מרחק הליכה** | השאירו עד כ־**800–1500 מ׳** | אם אין תוצאות — הגדילו את הסליידר |

חלופה לתל אביב: חיפוש לפי **שם תחנה** מה־GTFS, למשל `ת. רכבת יבנה` בשדה המוצא ויעד אחר על אותו קו (אחרי בחירה מהרשימה «— תחנת תח״צ»).

אם אין אף מסלול: נסו נקודות **קרובות יותר לתחנות** או **עיר עם כיסוי קווים גבוה** (מרכז הארץ לרוב עדיף לבדיקה ראשונה).

## מגבלות נתונים (למה לא «כל התחבורה הציבורית»)

בגלל **מגבלות אחסון וזמן ריצה** בפרויקט, **לא נטענו למסד כל נתוני התחבורה הציבורית** בצורה המלאה כמו בפיד GTFS המקורי של משרד התחבורה. בפרט:

- **אין טבלת `stop_times`** — לא הועלו מיליוני השורות מ־`stop_times.txt`, ולכן אין לוחות זמנים, תדירות או מסלולים רב־שלביים (החלפות).
- **`gtfs_shapes`** — לא כל קווי הארץ עם כל נקודות ה־shape; נטען תת־קבוצה (לפי מכסה שנקבעה בפרויקט), ומשם נגזרים קטעי מסלול על המפה.
- **`gtfs_stop_route`** — במקום לחשב מ־`stop_times`, הקישורים בין תחנות לקווים נבנו בגישה **מרחבית** (תחנה קרובה לציר ה־shape של הקו), עם טווח מטרים קבוע. זה מאפשר «קו ישיר אחד» אמין ברוב המקרים, אבל לא זהה ל־100% לנתוני המפעיל.

לכן הפיצ'ר מתאר **תכנון היוריסטי**: קו ישיר + הליכה + אומדן זמן (לא ניווט תפעולי מלא).

לפרטים טכניים נוספים ראו גם `docs/DECISIONS.md` (סעיף תכנון מסלול A→B).
